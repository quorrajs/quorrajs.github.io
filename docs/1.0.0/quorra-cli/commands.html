Writing Commands

Quorra, like Laravel, allows you to write your own commands which can be ran from CLI or code. Files are stored in app/console/commands folder. This is the basic structure:

'use strict';
const Command = App.command;

class Inspire extends Command {
    constructor() {
        super();
        // Command signature
        this.signature = 'greet {user}';
    }

    // Execute the command
    handle() {
        const user = this.argument('user');
        console.log('Hi, ', user);
    }
}

module.exports = Inspire;

In the constructor, after super() method, you have to fill a signature with your command name and its inputs. Then, in the handle() method you can write your code which will be execute when the command will be fired. You can find a complete example in app/console/commands/Inspire.js file.

Note: 'use strict'; is required since we are using ES6 classes feature.

Defining Input Expectations

When writing console commands, it is common to gather input from the user through arguments or options. Quorra makes it very convenient to define the input you expect from the user using the signature property on your commands. The signature property allows you to define the name, arguments, and options for the command in a single, expressive, route-like syntax.

Arguments

All user supplied arguments and options are wrapped in curly braces. In the following example, the command defines one required argument user:

this.signature = 'greet {user}';

You may also define default values for arguments:

this.signature = 'greet {user=foo}';

Options

Options, like arguments, are another form of user input. Options are prefixed by two hyphens (--) when they are specified on the command line. There are two types of options: those that receive a value and those that don't. Options that don't receive a value serve as a boolean "switch". Let's take a look at an example of this type of option:

this.signature = 'greet {user} {--hand}';

In this example, the --hand switch may be specified when calling the command. If the --hand switch is passed, the value of the option will be true. Otherwise, the value will be false:

quorra greet John --hand

Options with values

Next, let's take a look at an option that expects a value. If the user must specify a value for an option, suffix the option name with a = sign:

this.signature = 'greet {user} {--hand=}';

In this example, the user may pass a value for the option like so:

quorra greet John --hand=left

You may assign default values to options by specifying the default value after the option name. If no option value is passed by the user, the default value will be used:

this.signature = 'greet {user} {--hand=left}';

Input Arrays

If you would like to define arguments or options to expect array inputs, you may use the * character. First, let's take a look at an example that specifies an array argument:

this.signature = 'greet {user*}';

When calling this method, the user arguments may be passed in order to the command line. For example, the following command will set the value of user to ['John', 'Mike']:

quorra greet John Mike

When defining an option that expects an array input, each option value passed to the command should be prefixed with the option name:

this.signature = 'greet {user} {--hand*}';

quorra greet John --hand=left --hand=right

Command I/O

Retrieving Input

While your command is executing, you will obviously need to access the values for the arguments and options accepted by your command. To do so, you may use the argument and option methods:

// Execute the command
    handle() {
        const user = this.argument('user');
    }

If you need to retrieve all of the arguments as an object (dict), call the arguments method:

const args = this.arguments();

Options may be retrieved just as easily as arguments using the option method. To retrieve all of the options as an object, call the options method:

// Retrieve a specific option...
const hand = this.option('hand');

// Retrieve all options...
const options = this.options();

If the argument or option does not exist, null will be returned.

Writing Output

To send output to the console you can simply use the classic console.log() method. In future there will be methods to print colored text.

console.log('Hi, ', this.argument('user'));

Registering Commands

Once your command is finished, you need to register it with Quorra. All commands are registered in the app/console/kernel.js file. Within this file, you will find a list of commands in the commands property. To register your command, simply add the command's class name to the list:

'commands': [
    'app/console/commands/Inspire'
]

Call Commands From Your App Code

Sometimes you may wish to execute a command outside of the CLI. For example, you may wish to fire it from a route or controller. You may use the run method of the Command class to accomplish this. The run method accepts the name of the command as the first argument, and an object of command parameters as the second argument.

Note: If you need to pass a switch parameter (option without value), like --hand, set the value of the key to true. 

const Command = App.command;
Route.get('/', function(req, res){
    Command.run('greet', {
        user: ['John', 'Mike'], // Array input
        '--hand': 'left', // Option
        '--near': true // Switch
    });
});
